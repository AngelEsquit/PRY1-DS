Proyecto 1. 
Obtención y Limpieza de los datos 
 INTRODUCCIÓN:  
Los tipos y las fuentes de datos como se encuentran en las organizaciones son realmente diversos. 
El primer trabajo de un científico de datos es acceder a las fuentes y preparar los datos para que 
puedan ser analizados. Usualmente esto lleva un gran trabajo, pero sin hacerlo, el riesgo de llegar a 
resultados erróneos es demasiado alto . Con la realización de este proyecto, no solo aprenderá a 
utilizar las herramientas que le permitan acceder a los datos, sino a limpiarlos, haciéndolo de la 
forma más transparente posible. 
Competencias: 
- Utiliza las herramientas que tiene a su disposición para obtener los datos de la fuente 
especificada. 
- Modifica los datos que obtuvo realizando procesos de limpieza que allanen el camino del 
analista de datos. 
- Hace el proceso de limpieza transparente y reproducible para cualquiera que lo desee  
verificar. 
- Elabora un “Code Book” detallado que contenga todos los metadatos que sean necesarios 
para analizar los datos. 
ACTIVIDADES 
1. Descargue los datos de los establecimientos educativos de todo el país que lleguen hasta el 
nivel de diversificado (NIVEL ESCOLAR: DIVERSIFICADO). Los puede encontrar en el siguiente 
vínculo: http://www.mineduc.gob.gt/BUSCAESTABLECIMIENTO_GE/ 
2. Guarde los datos crudos en archivos .csv. 
3. Análisis del estado inicial de los datos : Elabore un diagnóstico del conjunto de datos que, 
como mínimo incluya: 
a. Número de registros y variables.  
b. Tipo de dato de cada variable.  
c. Cantidad y porcentaje de valores faltantes por variable.  
d. Cantidad de valores únicos.  
e. Cantidad de registros duplicados exactos.  
f. Variables con valores fuera de dominio o inconsistentes.  
g. Variables con formatos inconsistentes.  
h. Identificación de problemas potenciales de calidad de datos. 
Este análisis debe estar respaldado por tablas y estadísticas generadas usando código.

5. Limpieza del conjunto de datos: Realice todas las operaciones necesarias para obtener un

b. Verifique que cada variable tenga el tipo de dato apropiado.

• números almacenados como texto,  
• fechas almacenadas como cadenas,

7. Validación del conjunto limpio: Al finalizar la limpieza deberá ejecutar pruebas automáticas 
que verifiquen la calidad del conjunto de datos. Deberá comprobar que: 
• no existan registros duplicados exactos;  
• no existan espacios al inicio o final de textos;  
• los teléfonos tengan un formato consistente;  
• los nombres de departamentos y municipios pertenezcan al catálogo 
correspondiente;  
• las variables tengan el tipo de dato esperado;  
• no existan categorías duplicadas por diferencias de escritura;  
• no existan valores inválidos detectados durante el diagnóstico inicial.

En la que ponga la siguiente información: 
 
Registros: Número total de filas del conjunto de datos. Debe justificarse si cambió (por 
ejemplo, eliminación de duplicados exactos). 
Variables: Número total de columnas. Si se agregaron variables derivadas o se eliminaron 
columnas vacías, debe explicarse. 
Valores faltantes: Número y porcentaje total de celdas con valores faltantes (NA, NULL, 
cadenas vacías, etc.). 
Variables con NA: Cantidad de variables que contienen al menos un valor faltante. 
Duplicados exactos: Número de registros idénticos en todas sus columnas. 
Posibles duplicados: Número de registros identificados como posibles duplicados 
mediante similitud de nombres, direcciones o teléfonos. Debe indicarse cuántos fueron 
corregidos, fusionados o conservados. 
Variables con formato inconsistente: Cantidad de variables que presentaban problemas 
de formato (espacios, mayúsculas, teléfonos, fechas, etc.). 
Variables con tipo incorrecto: Cantidad de variables cuyo tipo de dato no coincidía con el 
esperado (por ejemplo, números almacenados como texto). 
Categorías inconsistentes: Número de categorías diferentes que representaban el mismo 
valor (por ejemplo, "Guatemala", "GUATEMALA", "guatemala"). 
Errores corregidos: Total de correcciones realizadas durante la limpieza, o un resumen por 
tipo de error.

todos los departamentos que cumpla las siguientes condiciones: 
a. estructura consistente;  
b. tipos de datos correctos;  
c. nombres de variables descriptivos;  
d. formato uniforme en todas las columnas;  
e. ausencia de errores detectados durante la fase de validación.

EVALUACIÓN 
- (5 puntos) Obtención y documentación y unión de los datos.  
- (15 puntos) Análisis del estado de los datos crudos : Se hace un análisis detallado del 
estado de los datos antes de comenzar las operaciones de limpieza. (Diagnóstico) 
- (10 puntos) Plan de limpieza. Se elabora el plan de limpieza  en el que especifican que 
problema tiene el campo, cual es la posible transformación y los riesgos de aplicarlas.  
- (30 puntos) Se hace una limpieza de calidad  sobre todas las variables  del conjunto de 
datos total, realizando todas las operaciones propuestas en esta guía. Se demuestra la 
capacidad para identificar registros repetidos tanto exactos como parciales. 
- (10 puntos) Se implementan pruebas que permitan verificar que el conjunto de datos 
cumple las reglas de calidad establecidas 
- (10 puntos) Se evalúa la capacidad para demostrar objetivamente la mejora obtenida 
después del proceso de limpieza. 
- (10 puntos)  Libro de códigos.  El libro de código debe ser comprensible, estar bien 
detallado y organizado. Contiene los metadatos necesarios para comprender el 
conjunto de datos, incluidas las fechas en las que se extrajeron y la fuente. 
- (10 puntos) Generación del conjunto de datos Limpios . Se genera un solo conjunto de 
datos con la información de todos los departamentos. Los datos generados están 
limpios y pueden utilizarse para hacer análisis.
